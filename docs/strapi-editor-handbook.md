# Понятная инструкция для редактора Strapi

Этот документ написан для человека, который не работает с кодом и хочет управлять сайтом через Strapi.

Если вы открыли Strapi впервые, начните с короткой карты интерфейса: [Карта интерфейса Strapi](strapi-ui-map.md).

Если нужно не просто понять поля, а увидеть примеры заполнения страниц, используйте: [Рецепты заполнения страниц](strapi-page-recipes.md).

## Что такое Page

`Page` в Strapi - это одна публичная страница сайта.

Примеры:

- `/pricing` - страница цен.
- `/promo` - промо-страница.
- `/compare/respond-io` - страница сравнения.
- `/channels/email/amocrm` - страница пересечения канала и интеграции.

Техническое имя этой сущности в коде - `page_v2`. В интерфейсе можно думать о ней просто как о странице.

## Где найти страницу

1. Откройте Strapi admin.
2. Слева нажмите `Content Manager`.
3. Откройте коллекцию `Page`.
4. В поиске введите адрес страницы, например `/pricing`.
5. Откройте найденную запись.

Если вы не знаете точный адрес, ищите по `title`.

## Какие поля важны

| Поле | Что означает | Когда менять |
| --- | --- | --- |
| `title` | внутреннее название страницы | можно менять для удобства |
| `route_path` | публичный адрес страницы | менять осторожно, это URL |
| `page_kind` | тип страницы: landing, comparison, resource и т.д. | при создании новой страницы |
| `blueprint` | набор правил для блоков | при создании или смене типа страницы |
| `sections` | блоки страницы в нужном порядке | основное место редактирования |
| `seo_title` | title для поисковиков | перед публикацией обязательно |
| `seo_description` | description для поисковиков | перед публикацией обязательно |
| `show_in_header` | показывать в верхнем меню | если нужна ссылка в шапке |
| `show_in_footer` | показывать в футере | если нужна ссылка внизу |
| `show_in_sitemap` | показывать в sitemap | почти всегда включено для публичных страниц |
| `nav_label` | текст ссылки в меню | если страница есть в навигации |
| `nav_group` | группа меню | чтобы ссылка попала в нужную колонку |
| `editorial_status` | draft, review, approved, archived | перед publish ставьте `approved` |

## Какие поля лучше не трогать без разработчика

Эти поля защищают старые страницы от поломки макета:

- `migration_ready`
- `parity_status`
- `legacy_template_family`
- `legacy_layout_signature`
- `parity_notes`

Если случайно включить `migration_ready` на старой странице без проверки, сайт может показать не тот макет. Если страница выглядит плохо, выключите `migration_ready` и сообщите route разработчику.

## Как изменить текст на странице

1. Откройте нужную `Page`.
2. Найдите поле `sections`.
3. Раскройте нужный блок.
4. Измените текстовые поля.
5. Нажмите `Save`.
6. Проверьте preview или локальную страницу.
7. Если всё хорошо, поставьте `editorial_status = approved`.
8. Нажмите `Publish`.

## Как добавить новый блок

1. Откройте страницу.
2. В поле `sections` нажмите `Add component`.
3. Выберите блок.
4. Заполните поля блока.
5. Перетащите блок в нужное место.
6. Нажмите `Save`.
7. Проверьте страницу перед publish.

Чаще всего используются:

- `hero` - первый экран страницы.
- `rich-text` - обычный текст.
- `cards-grid` - карточки преимуществ, сценариев или материалов.
- `feature-list` - список возможностей.
- `steps` - шаги процесса.
- `faq` - частые вопросы.
- `comparison-table` - таблица сравнения.
- `pricing-plans` - тарифы.
- `before-after` - блок "до / после".
- `internal-links` - ссылки на соседние страницы.
- `final-cta` - финальный призыв к действию.

## Как создать новую страницу

1. В коллекции `Page` нажмите `Create new entry`.
2. Заполните `title`.
3. Заполните `slug`, например `new-campaign`.
4. Заполните `route_path`, например `/new-campaign`.
5. Выберите `page_kind`.
6. Выберите `blueprint`.
7. Заполните `seo_title` и `seo_description`.
8. Добавьте блоки в `sections`.
9. Если нужна навигация, заполните `show_in_header`, `show_in_footer`, `nav_group`, `nav_label`, `nav_order`.
10. Если страница должна быть в sitemap, включите `show_in_sitemap`.
11. Поставьте `editorial_status = approved`.
12. Нажмите `Publish`.

Для новой страницы обычно не нужно трогать `migration_ready` и `parity_status`.

## Как убрать страницу с сайта

Для новой страницы:

1. Откройте страницу.
2. Нажмите `Unpublish`.

Для старой перенесённой страницы:

1. Откройте страницу.
2. Выключите `migration_ready`.
3. Сохраните.

Так сайт вернётся к legacy fallback, если он есть.

## Как понять, что страница готова

Страница готова, если:

- есть нормальный `route_path`;
- заполнены `seo_title` и `seo_description`;
- есть хотя бы один смысловой блок в `sections`;
- все CTA ведут на правильные ссылки;
- FAQ не содержит тестовых вопросов;
- `editorial_status = approved`;
- страница открывается в браузере и выглядит нормально.

## Что делать, если видите странный контент

Примеры плохих признаков:

- текст на английском там, где нужен русский;
- вопрос вроде `Can this page be edited in Strapi?`;
- пустая таблица;
- дубль блока "соседние сравнения";
- блок появился два раза;
- старый макет стал слишком коротким или потерял секции.

Что сделать:

1. Не публикуйте страницу дальше.
2. Если это старая страница, выключите `migration_ready`.
3. Запишите `route_path`.
4. Проверьте, какой блок содержит плохой текст.
5. Передайте route и название блока разработчику.

## Что насчет AI

Сейчас основной рабочий режим для редактора - ручное создание и редактирование страниц в `Page`.

AI-генерацию не используйте как обычный редакторский процесс, пока она отдельно не включена и не проверена. Когда AI появится в работе, он должен будет создавать такую же `Page`, как человек, но сначала как черновик.

Даже после включения AI редактор всё равно будет проверять:

- адрес страницы;
- SEO;
- выбранные блоки;
- тексты;
- ссылки;
- sitemap и навигацию.

AI не должен включать `migration_ready` для старых страниц без проверки.
