# Быстрый старт редактора

Этот документ для человека, который работает только в Strapi и не хочет разбираться в коде.

## Что важно понять за 1 минуту

- Сайт уже работает на сервере: `https://astro.integromat.ru`.
- Контент редактируется в Strapi: `https://strapi.integromat.ru/admin`.
- Страницы сайта хранятся в разделе `Page`.
- Каждая запись `Page` — это одна публичная страница сайта.
- Адрес страницы задается полем `route_path`, например `/pricing` или `/promo`.
- Контент страницы собирается из блоков в поле `sections`.
- Старые страницы защищены: если новая версия выглядит плохо, можно быстро вернуть старый вариант через `migration_ready = false`.

## Где искать страницу

1. Откройте админку Strapi.
2. В левом меню откройте `Content Manager`.
3. Найдите коллекцию `Page`.
4. Используйте поиск по `title` или `route_path`.

Пример: чтобы найти страницу цен, ищите `/pricing`.

## Как изменить текст на странице

1. Откройте нужную страницу в `page_v2`.
2. Найдите поле `sections`.
3. Откройте нужный блок.
4. Измените текст.
5. Нажмите `Save`.
6. Если страница готова к публикации, поставьте `editorial_status = approved`.
7. Нажмите `Publish`.

Если это уже перенесенная старая страница, она останется на своем старом макете. Мы меняем контент, а не ломаем шаблон.

## Как добавить новый блок

1. Откройте страницу.
2. В поле `sections` нажмите `Add component`.
3. Выберите нужный блок.
4. Заполните поля блока.
5. Перетащите блок в нужное место, если надо изменить порядок.
6. Нажмите `Save`.

Самые частые блоки:

- `hero` — первый экран страницы.
- `cards-grid` — карточки преимуществ, сценариев или разделов.
- `feature-list` — список возможностей.
- `steps` — шаги процесса.
- `faq` — вопросы и ответы.
- `comparison-table` — таблица сравнения.
- `before-after` — блок “до / после”.
- `internal-links` — ссылки на связанные страницы.
- `final-cta` — финальный призыв к действию.

## Как создать новую страницу

1. В `Page` нажмите `Create new entry`.
2. Заполните обязательные поля:
   - `title`
   - `slug`
   - `route_path`
   - `page_kind`
   - `blueprint`
   - `seo_title`
   - `seo_description`
3. Добавьте блоки в `sections`.
4. Если страница должна быть в меню, заполните:
   - `show_in_header`
   - `show_in_footer`
   - `nav_group`
   - `nav_label`
   - `nav_order`
5. Если страница должна попасть в sitemap, включите `show_in_sitemap`.
6. Поставьте `editorial_status = approved`.
7. Нажмите `Publish`.

Для новой страницы обычно не нужно трогать `migration_ready` и `parity_status`. Эти поля нужны для старых страниц, которые переносятся со старого макета.

## Как убрать страницу с сайта

Если это новая страница без старого fallback:

1. Откройте страницу.
2. Нажмите `Unpublish`.

Если это старая перенесенная страница:

1. Откройте страницу.
2. Выключите `migration_ready`.
3. Нажмите `Save` и `Publish`, если Strapi просит обновить опубликованную версию.

После этого сайт снова покажет старый fallback-макет.

## Какие поля не трогать без проверки

Не меняйте без технической проверки:

- `migration_ready`
- `parity_status`
- `legacy_template_family`
- `legacy_layout_signature`
- `parity_notes`

Эти поля отвечают за безопасный перенос старой страницы. Ошибка здесь может переключить страницу не на тот макет.

## Что делать, если страница выглядит плохо

1. Не удаляйте страницу сразу.
2. Для старого маршрута выключите `migration_ready`.
3. Проверьте сайт еще раз.
4. Если нужно полностью скрыть новую запись, нажмите `Unpublish`.
5. Сообщите техническому оператору route, например `/pricing`.

## Простая памятка

- Нужно изменить текст — редактируйте блоки в `sections`.
- Нужно добавить страницу — создайте новую запись `Page`, заполните SEO и блоки, поставьте `editorial_status = approved`, нажмите `Publish`.
- Нужно добавить в меню — настройте `show_in_header`, `show_in_footer`, `nav_group`, `nav_label`.
- Нужно убрать из sitemap — выключите `show_in_sitemap`.
- Старая страница сломалась после переноса — выключите `migration_ready`.
