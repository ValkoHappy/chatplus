# Рецепты ручного создания страниц в Strapi

Этот документ нужен редактору, который хочет вручную создать или изменить страницу без кода. Здесь нет AI-сценариев: сейчас считаем, что страницы собирает человек в Strapi.

Главная идея простая: одна запись `Page` в Strapi = одна публичная страница сайта. Внутри записи задаются URL, SEO, меню, sitemap, хлебные крошки, блоки и финальная публикация.

## Быстрый маршрут

1. Откройте Strapi.
2. Нажмите `Content Manager`.
3. Откройте коллекцию `Page`.
4. Для новой страницы нажмите `Create new entry`.
5. Для изменения старой страницы найдите ее по `route_path`, например `/pricing`.
6. Заполните базовые поля страницы.
7. Соберите блоки в `sections`.
8. Проверьте страницу на сайте.
9. Поставьте `editorial_status = approved`.
10. Нажмите `Publish`.

## Что обязательно заполнить

Эти поля нужны почти всегда. Если их не заполнить, страница может не открыться, не попасть в sitemap или плохо выглядеть в поиске.

| Поле | Что писать | Пример |
| --- | --- | --- |
| `title` | Внутреннее название страницы для редактора | Промо для агентств |
| `slug` | Короткий ключ без пробелов | agency-promo |
| `route_path` | Публичный URL, всегда начинается с `/` | /agency-promo |
| `locale` | Язык страницы | ru |
| `page_kind` | Тип страницы | campaign |
| `blueprint` | Правила блоков для этого типа страницы | campaign |
| `template_variant` | Вариант отображения | showcase |
| `generation_mode` | Как создана страница | manual |
| `source_mode` | Источник страницы | managed |
| `seo_title` | Заголовок для поиска и вкладки браузера | Chat Plus для digital-агентств |
| `seo_description` | Короткое описание для поиска | Добавьте омниканальные чаты и AI-ответы в пакет услуг агентства. |
| `robots` | Индексация | index,follow |
| `show_in_sitemap` | Показывать в sitemap | true |
| `editorial_status` | Готовность редакции | draft, потом approved |

## Что не трогать без причины

Эти поля защищают старые страницы от поломки макета. Для новой страницы обычно не нужно их менять.

| Поле | Почему осторожно |
| --- | --- |
| `migration_ready` | Включает замену старого URL новой записью. На старых страницах включать только после проверки макета. |
| `parity_status` | Статус совпадения со старым макетом. Для старых страниц нужен `approved`, для новой страницы обычно можно оставить как есть. |
| `legacy_template_family` | Семейство старого шаблона. Нужен разработчикам и мосту миграции. |
| `legacy_layout_signature` | Технический снимок старого макета. |
| `parity_notes` | Технические заметки проверки переноса. |

Правило безопасности: если вы создаете абсолютно новую страницу на новом URL, обычно не трогайте migration-поля. Если вы редактируете старый URL, не включайте `migration_ready`, пока визуально не проверили страницу.

## Как выбрать тип страницы

| Что создаем | `page_kind` | `blueprint` | Обычно подходит |
| --- | --- | --- | --- |
| Промо, акция, рекламный лендинг | campaign | campaign | /agency-promo |
| Обычный коммерческий лендинг | landing | landing | /crm-automation |
| Страница компании, команды, медиа, бренда | brand | brand | /about-product |
| Документация, помощь, блог, статья | resource | resource | /docs/whatsapp-crm |
| Каталог разделов | directory | directory | /new-directory |
| Страница одной сущности | entity_detail | entity_detail | /features/new-feature |
| Пересечение двух сущностей | entity_intersection | entity_intersection | /channels/telegram/retail |
| Сравнение с конкурентом | comparison | comparison | /compare/new-tool |
| Служебная страница | system | system | /legal |

Если сомневаетесь, для ручной маркетинговой страницы чаще всего выбирайте `campaign` или `landing`.

## Как выбрать блоки

Блоки находятся в поле `sections`. Порядок блоков в Strapi = порядок блоков на странице.

| Блок | Для чего нужен | Когда использовать |
| --- | --- | --- |
| `hero` | Первый экран страницы | Почти всегда первым блоком |
| `rich-text` | Обычный текстовый раздел | Статьи, инструкции, пояснения |
| `proof-stats` | Цифры, факты, короткие доказательства | После hero или перед CTA |
| `cards-grid` | Карточки преимуществ, сценариев, материалов | Самый универсальный блок |
| `feature-list` | Список возможностей | Когда нужно перечислить функции |
| `steps` | Пошаговый процесс | Подключение, внедрение, workflow |
| `faq` | Частые вопросы | Почти всегда ближе к концу |
| `testimonial` | Цитата или отзыв | Когда есть социальное доказательство |
| `pricing-plans` | Тарифы | Только для pricing/offer страниц |
| `comparison-table` | Таблица сравнения | Для comparison, pricing, partnership |
| `before-after` | До/после, проблема/решение, ROI | Для partnership, pricing, лендингов |
| `internal-links` | Что еще посмотреть | Ближе к концу страницы |
| `related-links` | Связанные страницы | Для каталогов и SEO-страниц |
| `final-cta` | Финальный призыв к действию | Последний или предпоследний блок |

## Минимальная рабочая страница

Если нужно быстро создать простую страницу, достаточно такого набора:

1. `hero`
2. `cards-grid`
3. `faq`
4. `internal-links`
5. `final-cta`

Этого хватает, чтобы страница имела первый экран, смысловые блоки, вопросы, перелинковку и действие.

## Как настроить меню

Меню не появляется само по себе. Его нужно включить осознанно.

| Поле | Что делает | Пример |
| --- | --- | --- |
| `show_in_header` | Показывает ссылку в верхнем меню | true |
| `show_in_footer` | Показывает ссылку в футере | true |
| `nav_group` | Группа меню | product |
| `nav_label` | Текст ссылки | CRM-автоматизация |
| `nav_description` | Короткое описание | Сценарии CRM и передачи лидов |
| `nav_order` | Порядок внутри группы | 40 |

Группы:

| `nav_group` | Где показывается |
| --- | --- |
| `product` | Продукт |
| `catalogs` | Каталоги |
| `resources` | Ресурсы |
| `company` | Компания |
| `special` | Спецразделы |
| `primary` | Верхняя навигация |

Если страница экспериментальная, не включайте `show_in_header`. Лучше добавить ее в `internal-links` на связанных страницах или только в footer.

## Как настроить sitemap

| Поле | Рекомендация |
| --- | --- |
| `show_in_sitemap` | true для публичных страниц |
| `sitemap_priority` | 0.5 по умолчанию, 0.8 для важных коммерческих страниц |
| `sitemap_changefreq` | weekly для обычных страниц, monthly для редко меняющихся |
| `robots` | index,follow для публичных страниц |

Не ставьте `show_in_sitemap=true`, если страница тестовая, временная или не должна индексироваться.

## Как настроить хлебные крошки

Для вложенных страниц используйте `parent_page`.

Примеры:

| Страница | `parent_page` |
| --- | --- |
| /compare/respond-io | /compare |
| /docs/whatsapp-crm-guide | /docs |
| /features/ai-calendar | /features |

Если автоматической иерархии не хватает, заполните `breadcrumbs` вручную:

| `title` | `href` |
| --- | --- |
| Главная | / |
| Сравнения | /compare |
| Respond.io | /compare/respond-io |

## Как проверить ссылки

Перед публикацией откройте все CTA и internal links.

Проверить нужно:

- `primary_url`;
- `secondary_url`;
- ссылки внутри `internal-links`;
- ссылки внутри карточек;
- ссылки в final CTA;
- ссылки в breadcrumbs.

Не добавляйте ссылку на текущую же страницу. Например, на `/compare/respond-io` не нужна карточка `/compare/respond-io` в блоке `internal-links`.

## Где в Strapi задаются ссылки кнопок

Да, ссылки кнопок хранятся в Strapi. У каждой кнопки обычно есть два поля: текст кнопки и URL.

Важно: в текущих page-blocks поля называются `*_url`, а не `*_href`. Например, в `hero` это `primary_url`, а в `internal-links` это уже `href`.

| Где кнопка или ссылка | Поле текста | Поле ссылки | Пример |
| --- | --- | --- | --- |
| Главная кнопка первого экрана `hero` | `primary_label` | `primary_url` | `Попробовать бесплатно` -> `/demo` |
| Вторая кнопка первого экрана `hero` | `secondary_label` | `secondary_url` | `Посмотреть цены` -> `/pricing` |
| Главная кнопка финального CTA `final-cta` | `primary_label` | `primary_url` | `Записаться на демо` -> `/demo` |
| Вторая кнопка финального CTA `final-cta` | `secondary_label` | `secondary_url` | `Посмотреть тарифы` -> `/pricing` |
| Кнопка в тарифе `pricing-plans` | `cta_label` | `cta_url` | `Выбрать план` -> `/demo` |
| Карточка в `cards-grid` | `cta_label` | `href` | `Смотреть раздел` -> `/features` |
| Ссылка в `internal-links` | `label` | `href` | `Цены` -> `/pricing` |
| Ссылка в `related-links` | `label` | `href` | `Интеграции` -> `/integrations` |
| Хлебная крошка `breadcrumbs` | `label` | `href` | `Сравнения` -> `/compare` |

Если на странице есть кнопка `Демо`, `Попробовать`, `Записаться`, `Рассчитать`, почти всегда ее URL лежит рядом в том же блоке.

## Рекомендуемые адреса для CTA

Чтобы кнопки вели предсказуемо, используйте единые адреса.

| Кнопка | Куда обычно вести |
| --- | --- |
| `Попробовать бесплатно` | `/demo` |
| `Записаться на демо` | `/demo` |
| `Получить демо` | `/demo` |
| `Обсудить внедрение` | `/demo` |
| `Рассчитать стоимость` | `/pricing` или `/demo` |
| `Посмотреть цены` | `/pricing` |
| `Партнерам` | `/partnership` |
| `Сравнить альтернативы` | `/compare` |
| `Смотреть интеграции` | `/integrations` |
| `Смотреть каналы` | `/channels` |

Для внутренних страниц используйте короткий внутренний URL без домена: `/demo`, `/pricing`, `/compare/respond-io`.

Для внешней ссылки можно использовать полный URL: `https://example.com/page`.

## Как проверить, куда ведет кнопка

1. Откройте `Content Manager -> Page`.
2. Найдите страницу по `route_path`.
3. Откройте `sections`.
4. Найдите блок, где находится кнопка: чаще всего `hero`, `final-cta`, `pricing-plans`, `cards-grid`.
5. Проверьте поле рядом с текстом кнопки:
   - для `hero` смотрите `primary_url` и `secondary_url`;
   - для `final-cta` смотрите `primary_url` и `secondary_url`;
   - для тарифа смотрите `cta_url`;
   - для карточки смотрите `href`;
   - для блока ссылок смотрите `href`.
6. Откройте этот URL в браузере и убедитесь, что страница существует.

Если кнопка есть, а URL пустой, кнопка может не появиться на сайте или будет выглядеть сломанной.

## Частые ошибки в ссылках кнопок

| Ошибка | Почему плохо | Как правильно |
| --- | --- | --- |
| `demo` без слеша | Это не нормальный внутренний route | `/demo` |
| `/demo/` со слешем в конце | На сайте есть редиректы, лучше не плодить варианты | `/demo` |
| Пустой `primary_url` при заполненном `primary_label` | Кнопка может не отрендериться | Заполнить URL или убрать label |
| Ссылка на текущую же страницу | Пользователь кликает и остается там же | Дать соседнюю полезную страницу |
| Внутренняя ссылка с полным доменом | Сложнее переносить между локалкой/сервером | `/pricing`, а не `https://site.ru/pricing` |
| Внешняя ссылка без `https://` | Браузер может считать ее внутренней | `https://...` |

## Как удалить или спрятать страницу

Для новой страницы:

1. Откройте `Page`.
2. Нажмите `Unpublish`.
3. Если страница не нужна совсем, после проверки можно удалить запись.

Для старой перенесенной страницы:

1. Сначала выключите `migration_ready`.
2. Сохраните.
3. Проверьте, что сайт вернулся на legacy fallback.
4. Только потом решайте, нужно ли снимать publish.

Не удаляйте legacy template из кода вручную. Старые шаблоны пока нужны как страховка.

## Проверка перед Publish

Ответьте `да` на каждый пункт:

- URL в `route_path` правильный и начинается с `/`.
- Нет лишнего `/` в конце URL.
- `seo_title` заполнен.
- `seo_description` заполнен.
- `page_kind` выбран правильно.
- `blueprint` выбран правильно.
- В `sections` есть `hero`.
- В `sections` есть смысловой блок после hero.
- В `sections` есть CTA или понятный следующий шаг.
- FAQ не содержит английских заглушек.
- Таблицы не пустые.
- Карточки не пустые.
- Ссылки ведут на существующие страницы.
- Страница не дублирует текущий URL в internal links.
- Меню включено только если страница должна быть в меню.
- Sitemap включен только если страница должна индексироваться.
- Для старого URL `migration_ready` не включали без визуальной проверки.

## Частые ошибки

| Ошибка | Что будет | Как исправить |
| --- | --- | --- |
| `route_path` без `/` | Страница может не открыться | Написать `/my-page` |
| Пустой `seo_description` | Плохо для поиска и publish guardrails | Написать 1-2 предложения |
| Слишком много ссылок в header | Шапка ломается или становится шумной | Оставить header только для главных разделов |
| Английские заглушки в FAQ | На сайте видно тестовый текст | Заменить на реальные вопросы |
| Пустые строки в таблице | Таблица выглядит поломанной | Заполнить или удалить строку |
| Повтор текущей страницы в internal links | Появляется дублирующая карточка | Удалить ссылку на текущий URL |
| Включили `migration_ready` на старой странице без проверки | Старый макет может замениться неполной версией | Выключить `migration_ready` |

## Что делать, если страница выглядит плохо

1. Запишите `route_path`.
2. Откройте эту `Page` в Strapi.
3. Проверьте `sections`: нет ли пустых блоков, дублей, английских заглушек.
4. Проверьте `internal-links`: нет ли ссылки на текущую страницу.
5. Если это старый URL, выключите `migration_ready`.
6. Сохраните и проверьте сайт снова.
7. Если проблема в макете, передайте разработчику `route_path` и название блока.
