# Глоссарий

## Strapi

Админка и CMS, в которой хранятся страницы, блоки, SEO и служебные сущности сайта.

## Page

Публичная страница сайта. У неё есть адрес, заголовки, блоки и настройки показа.

## Block

Отдельная секция страницы. Например hero, FAQ, карточки или CTA.

## Blueprint

Стартовая схема страницы. Она подсказывает, какие блоки подходят для выбранного типа страницы.

## Draft

Черновик. Страница сохранена в Strapi, но ещё не опубликована для сайта.

## Review

Статус проверки. Значит страницу подготовили и отправили на просмотр.

## Approved

Статус одобрения. Значит страницу можно публиковать или готовить к переключению маршрута.

## Publish

Публикация записи в Strapi. Для новых страниц этого обычно достаточно, а для старых маршрутов ещё нужен safe cutover.

## Migration

Перенос старого маршрута на новый page layer в Strapi.

## Fallback

Запасной старый рендер. Если новая страница не готова, сайт показывает старую версию.

## Rollback

Быстрый возврат на старый рендер или на предыдущую рабочую версию.

## Sitemap

Карта сайта. Она помогает поисковикам понимать, какие страницы должны индексироваться.

## SEO

Поля для поисковиков: заголовок, описание, canonical и другие настройки страницы.

## Canonical

Главный адрес страницы для поисковиков. Нужен, чтобы не было дублей.

## Noindex

Запрет на индексацию страницы поисковиками.

## AI draft

Черновик страницы, который создала нейросеть. Он не должен публиковаться автоматически.
